HTML

HTML and the Web

HTML stands for Hypertext Markup Language. It is made up of text content (what you see), markup (what it looks like), and references to other documents (e.g. images).

The web works by a person going to a web page like Google. The computer sends a HTTP request to a server. The server finds the appropriate HTML document and sends back to the user's computer where the web browser interprets the page and displays it on the user's eescreen.

Tags and Elements

HTML documents are made of HTML elements. HTML elements are made up of an opening tag, contents, and closing tag. When writing HTML, we tell browsers the type of each element by using HTML tags.

A place were we can play around with how our markup language appears with is the Udacity Playground.

DOM stands for Document Object Model. DOM is a standard convention for representing and interacting with elements in HTML. Each HTML tag creates an element in the DOM that the browser uses to display the page.

Computers are Stupid

Computers are stupid. Humans tell computers what to do, so we have to be careful about mistakes.

What is the difference between Inline and Block Elements?

Inline elements include <br> and <em> tags which apply to the content inside that tag. Block elements are used to create "invisible boxes" around the content inside of them. Examples of a block element are <div> and <p>.

Version Control

Types of version control are: Manual Saving, Dropbox, Google Docs, Wikipedia, Git, and SVN. We use Git for the Udacity class.

Git is a version control system. Git allows saving multiple files in one commit, while others, like Google Docs, treat each file separately.

It's easier to break sections up and update as you work.

Avoiding Repetition

Repetition only leaves room for errors, especially when you have to repeat it in several different places. The best way is to assign a class for consistency in what you are wanting to repeat. This can be done via a style sheet.

CSS

CSS stands for cascading style sheets. It is a style sheet language that describes the look and format of a document written in HTML.
